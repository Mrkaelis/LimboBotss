# LimboBots v2.0.0

Плагин для Spigot/Paper **1.16.5**, добавляющий фейковых игроков (ботов) на сервер.
Автор: **MrSteve**

## Возможности

- Боты в **табе**, ниже реальных игроков (сортировка через фейковые scoreboard-команды).
- Боты в мире как **NPC-игроки** (модель игрока).
- Настраиваемый **префикс** в табе и чате: `%vcmute_status% %luckperms-prefix% %bot_name%`.
- Боту можно назначить **привилегию (LuckPerms-группу)** — влияет на префикс и на порядок сортировки среди других ботов.
- **"Живые" ники** ботов, генерируемые из пула обычных имён (не `Bot_1`, `Bot_2`).
- Плавная **прогулка** ботов маленькими шагами в радиусе их точки спавна, без "провала" под карту.
- Простая имитация **выживания**: боты **копают ресурсы**, иногда **дерутся друг с другом**, **умирают** и **автоматически возрождаются**.
- Периодические сообщения ботов в **чат** в том же формате префикса.
- Обычная команда **`/tp`** понимает имена ботов как цель.
- Плейсхолдер **`%limbobots_online%`** для PlaceholderAPI (реальный онлайн + боты).

## Команды `/limbobots` (права: `limbobots.admin`, по умолчанию OP)

| Команда | Описание |
|---|---|
| `/limbobots add <количество>` | Добавить N ботов |
| `/limbobots remove <количество>` | Удалить N ботов |
| `/limbobots clear` | Удалить всех ботов |
| `/limbobots list` | Список ботов (статус, ранг, HP) |
| `/limbobots tp <имя>` | Телепортироваться к боту |
| `/limbobots equip <имя> <слот> <предмет>` | Надеть предмет на бота |
| `/limbobots rank <имя> <группа_luckperms>` | Назначить боту привилегию |
| `/limbobots ranks` | Список групп LuckPerms |
| `/limbobots mute` / `unmute <имя>` | Переключить косметический статус "мута" |

Алиасы: `/lb`, `/bots`. Слоты для `equip`: `head`, `chest`, `legs`, `feet`, `hand`, `offhand`.

Дополнительно: обычная **`/tp <игрок/бот>`** и **`/tp <игрок> <игрок/бот>`** тоже работают с ботами
(см. раздел про `tp-override` ниже).

## Конфигурация (`config.yml`)

Все ключевые вещи вынесены в конфиг, чтобы не пересобирать плагин ради тонкой настройки:

```yaml
tab-format: " %vcmute_status% &f%luckperms-prefix%&7 %bot_name%"
vcmute: { muted-symbol: "&c[X]", unmuted-symbol: "", muted-chance: 0.15 }
names: { pool: [...], number-suffix-chance: 0.6 }
movement: { radius: 12, decide-interval-ticks: 60, step-size: 0.18 }
chat: { enabled: true, interval-ticks: 200, messages: [...] }
combat: { enabled: true, max-hp: 20, pvp-radius: 6, respawn-delay-seconds: 8, action-interval-ticks: 100, pvp-chance: 0.3 }
mining: { enabled: true, radius: 6, blocks: [...] }
tp-override: { enabled: true }
```

Полный файл с комментариями — в `src/main/resources/config.yml`.

## Как это работает (кратко)

### 1) Приоритет в табе (боты ниже игроков) + сортировка по привилегии
Ванильный клиент сортирует таб-лист по имени scoreboard-команды игрока
(без команды = пустая строка = сортируется первым), а затем по нику внутри
команды. Реальным игрокам мы не назначаем никакую команду — они всегда
идут первыми. Каждому боту присваивается **фейковая** (существующая только
на уровне пакетов, не в реальном скорборде сервера) команда с именем вида
`~<вес>_<id>` — символ `~` гарантированно "тяжелее" любых обычных букв/цифр,
поэтому боты всегда оказываются ниже игроков, а `<вес>` (из LuckPerms-группы)
определяет порядок между самими ботами.

### 2) Привилегии ботов
`/limbobots rank <бот> <группа>` **не создаёт** боту реального пользователя
в LuckPerms (это замусорило бы базу случайными UUID) — плагин просто читает
**вес и префикс уже существующей группы** и использует их для сортировки/отображения.

### 3) Движение без "провалов" и рывков
Раньше бот двигался от своей *текущей* точки, поэтому случайные шаги могли
накопиться и увести его за пределы платформы спавна (в пустоту). Теперь
цель для прогулки всегда считается от **исходной точки спавна бота**
(`homeLocation`) в пределах `movement.radius`, а перед тем как идти,
плагин проверяет, что в целевой точке вообще есть твёрдый блок и что
высота не отличается больше чем на 3 блока от исходной (иначе точка
считается обрывом/пустотой и пропускается). Само перемещение разбито на
маленькие шаги (`movement.step-size`) и рассылается обычными пакетами
относительного движения (как у настоящих ходящих игроков), а не одним
резким телепортом — это выглядит как нормальная ходьба, а не "полёт".

### 4) "Живые" ники
Вместо `Bot_1`, `Bot_2` — имена собираются из пула обычных ников
(`names.pool` в конфиге) с шансом добавить цифровой суффикс, как у реальных
игроков (`Nikita228`, `Shadow77` и т.п.). Пул полностью настраиваемый.

### 5) Обычный `/tp`
Плагин через рефлексию подменяет команду `tp`/`teleport` в `CommandMap`
сервера на свою реализацию, которая сначала ищет бота с таким именем,
и если не находит — ищет обычного игрока. Поддерживает:
`/tp <цель>`, `/tp <игрок> <цель>`, `/tp <x> <y> <z>`, `/tp <игрок> <x> <y> <z>`.
**Ограничение:** это упрощённая замена, она не поддерживает селекторы целей
(`@a`, `@e`, `@p`) и телепортацию с учётом направления взгляда/вида ванильной
команды. Если это критично — отключите `tp-override.enabled: false` в
конфиге и пользуйтесь безопасной версией `/limbobots tp <имя>`.

### 6) `%limbobots_online%`
Регистрируется в PlaceholderAPI (если он установлен) как расширение `limbobots`:
- `%limbobots_online%` — реальный онлайн + количество ботов
- `%limbobots_real_online%` — только реальные игроки
- `%limbobots_bots%` — только количество ботов

### 7) "Выживание": добыча, PvP, смерть, возрождение
Раз в `combat.action-interval-ticks` тиков каждый бот (если он никуда не идёт)
с шансом `combat.pvp-chance` атакует ближайшего другого бота в радиусе
`combat.pvp-radius` (анимация взмаха + случайный урон 1-4), либо пытается
сломать случайный подходящий блок в радиусе `mining.radius` из списка
`mining.blocks` — блок реально ломается через стандартный `Block#breakNaturally()`,
поэтому предметы выпадают как обычно. При HP ≤ 0 бот проигрывает анимацию
смерти, исчезает и через `combat.respawn-delay-seconds` секунд возрождается
на своей исходной точке спавна с полным HP.

**Важно про добычу ресурсов:** `breakNaturally()` — это прямое изменение
мира и **не вызывает** событие `BlockBreakEvent`, поэтому плагины защиты
территории (WorldGuard и т.п.) не смогут его перехватить. Рекомендуется
либо держать ботов в контролируемой зоне (спавн-платформа без ценных
построек рядом), либо ограничить список `mining.blocks` безопасными блоками,
либо выключить `mining.enabled: false`, если это нежелательно на вашем сервере.

**Также важно:** это НЕ полноценный ИИ (нет пути через препятствия, нет
настоящего инвентаря бота, PvP только "бот против бота", а не "бот против
игрока" — атака реальными игроками фейковых сущностей потребовала бы
перехвата сырых пакетов через инъекцию в Netty-канал, что не реализовано
в этой версии).

### 8) Формат префикса в чате и в табе
Оба места используют один и тот же настраиваемый формат `tab-format`
из конфига. Для таба строка автоматически разбивается на "префикс" (всё
до `%bot_name%`) и "суффикс" (всё после) и уходит в scoreboard-команду —
само имя в табе рисует клиент отдельно. Для чата подставляются все
плейсхолдеры целиком, и получившаяся строка используется как
`<Префикс> Имя: сообщение`.

**Про `%vcmute_status%`:** у ботов нет настоящего голосового чата, поэтому
это чисто **косметическая имитация** — случайный статус выставляется при
создании бота (`vcmute.muted-chance`) и переключается вручную командой
`/limbobots mute|unmute`. Если на сервере стоит реальный войсчат-плагин
с собственным PlaceholderAPI-плейсхолдером `%vcmute_status%`, ваш формат
в `tab-format` может использовать как этот плейсхолдер (для настоящих
игроков через сторонние плагины таба), так и наш (для ботов) — они не
конфликтуют, так как обрабатываются разными системами.

## Сборка проекта

```
java -jar BuildTools.jar --rev 1.16.5     # соберёт org.spigotmc:spigot:1.16.5-R0.1-SNAPSHOT в ~/.m2
mvn clean package                          # соберёт LimboBots.jar в target/
```

LuckPerms API и PlaceholderAPI подключены как `provided`-зависимости
(нужны только для компиляции, на сервере не обязательны — при их
отсутствии соответствующий функционал просто мягко отключается).

## Troubleshooting

### `InvalidDescriptionException: Invalid plugin.yml` / `Jar does not contain plugin.yml`

Это значит, что файл `plugin.yml` физически отсутствует внутри собранного
`LimboBots.jar` (не ошибка формата YAML — файла нет в архиве вообще).
Порядок проверки:

1. **Проверьте, что файл вообще закоммичен в репозиторий:**
   ```
   git ls-files | grep plugin.yml
   ```
   Если пусто — файл не в git. Самая частая причина — в репозитории уже
   был какой-то общий `.gitignore` (например, скопированный из другого
   проекта) с правилом вида `*.yml` или игнорирующий папку `resources/`.
   В этом архиве лежит безопасный `.gitignore`, который таких правил не содержит.

2. **Проверьте лог сборки на GitHub Actions** — в `build.yml` уже добавлены
   два диагностических шага:
   - "Verify project structure" — падает с понятной ошибкой, если
     `src/main/resources/plugin.yml` отсутствует в репозитории ещё до сборки;
   - "Verify plugin.yml is inside the built jar" — после сборки распаковывает
     `target/LimboBots.jar` и явно проверяет наличие `plugin.yml` внутри,
     печатая полный список файлов jar для диагностики.

3. **Если оба шага прошли на CI, но хостинг всё равно ругается** — скорее
   всего на хостинг был залит не тот файл (старый jar, файл повреждён при
   загрузке, и т.п.). Скачайте артефакт `LimboBots` заново со свежего запуска
   Actions и перезалейте именно его.

## Известные ограничения

- Скин ботов по умолчанию (Steve/Alex) — UUID случайный, не привязан к Mojang-аккаунту.
- Клик по боту (ПКМ/ЛКМ в мире) не ловится — используйте команды/`\tp`.
- PvP только между ботами, реальные игроки не могут атаковать ботов напрямую.
- `breakNaturally()` не проходит через `BlockBreakEvent` — не защищено WorldGuard-подобными плагинами.
- `%vcmute_status%` — не связан с реальным голосовым чатом, чисто косметическая имитация.
- Часть NMS-сигнатур (особенно `PacketPlayOutScoreboardTeam`) проверена по памяти о типичной
  структуре пакетов CraftBukkit 1.16.x, а не скомпилирована локально (в песочнице нет доступа
  к сети для скачивания реального spigot-jar). Если сборка на вашем CI покажет ошибку компиляции —
  пришлите лог, поправим по месту, как и раньше.
